欢迎使用由清弦凝绝开发的spine动画导出小工具(°?°)? 	<-控制台中的

1、使用 java -jar AnimationBuilder.jar 运行【请不要直接运行，在启动窗口前会有一些控制台事件】
2、-----------设置内容-----------
	a)设置屏幕的尺寸-【默认为1024X1024】
	b)设置渲染时的背景颜色-【默认为[0,255,0,255](RGBA)】
	c)设置FPS-【默认为30帧】
	d)设置屏幕标题-【默认为：Spine Animation Builder】
	e)设置是否使用openGL渲染-【默认为使用】
	f)设置spine的缩放-【默认为 1.0f】
	g)跳过设置
当需要设置的已经修改完毕后，使用“g”退出设置。退出时会打印所有可用的设置，如下
-----------设置内容-----------
	a)设置屏幕的尺寸-【为1024X1024】
	b)设置渲染时的背景颜色-【为[0.00,1.00,0.00,1.00](RGBA)[0.0,1.0]】
	c)设置FPS-【为30帧】
	d)设置屏幕标题-【为：Spine Animation Builder】
	e)设置是否使用openGL渲染-【为true】
	f)设置spine的缩放-【为 1.00】
随后会启动窗口。
3、在窗口中，使用“ctrl+L”打开载入skeleton文件的弹窗，选择文件格式为 *.skel 或者 *.skel.txt即可。
	程序会自动寻找atlas文件，如果找不到，将会弹出选择atlas文件的弹窗（atlas和texture必须在同一目录）（感觉atlas文	件的第一行是定义texture的相对路径，但是没有试过），选择“*.atlas”或“.atlas.txt”文件即可。
4、加载完成后，会打印所有自带的动画，可以输入动画名或编号进行循环播放（开始有2s延时）。（请不要输入错误的索引）
5、在播放状态下，使用“Ctrl +S”导出序列图。
------------------------------------
自定义动画挺麻烦的，如果是单纯地旋转骨骼，会让动画在基于旋转后的状态进行渲染。单纯的更换插件倒是不麻烦。
一个Animation实例需要timeLine，timeLine实例好像无法自定义。可能想要实现自定义动画，需要进行手动渲染，在渲染的时候添加自定义动画。
